# 前端环境变量配置说明

## 📋 配置文件说明

项目已配置多环境支持，包含以下环境变量文件：

- **`.env.development`** - 开发环境配置（运行 `npm run serve` 时自动加载）
- **`.env.production`** - 生产环境配置（运行 `npm run build` 时自动加载）

## 🔧 环境变量说明

### 开发环境 (.env.development)

```env
VUE_APP_API_BASE_URL=http://localhost:9090
VUE_APP_ENV=development
VUE_APP_DEBUG=true
```

- **API地址**：使用完整URL `http://localhost:9090`，通过 `vue.config.js` 的 proxy 代理
- **调试日志**：启用，会在控制台显示环境配置信息

### 生产环境 (.env.production)

```env
VUE_APP_API_BASE_URL=
VUE_APP_ENV=production
VUE_APP_DEBUG=false
```

- **API地址**：空字符串，使用相对路径，由 Nginx 反向代理
- **调试日志**：关闭

## 🚀 使用方法

### 开发环境

```bash
# 运行开发服务器（自动使用 .env.development）
npm run serve
```

启动后，浏览器控制台会显示：
```
=== 前端环境配置 ===
环境: development
API地址: http://localhost:9090
```

### 生产环境

```bash
# 打包生产版本（自动使用 .env.production）
npm run build

# 或者使用明确的命令
npm run build:prod
```

打包后的 `dist` 目录中的代码会使用相对路径 `/api`，由 Nginx 代理到后端。

### 可选：使用开发环境配置打包

```bash
# 使用开发环境配置打包（用于测试）
npm run build:dev
```

## 📝 代码修改说明

### 1. main.js

已修改为使用环境变量：

```javascript
// 从环境变量读取API地址
axios.defaults.baseURL = process.env.VUE_APP_API_BASE_URL || '';

// 开发环境显示配置信息
if (process.env.NODE_ENV === 'development' && process.env.VUE_APP_DEBUG === 'true') {
  console.log('=== 前端环境配置 ===');
  console.log('环境:', process.env.VUE_APP_ENV || 'development');
  console.log('API地址:', axios.defaults.baseURL || '相对路径（Nginx代理）');
}
```

### 2. vue.config.js

已修改 proxy 配置使用环境变量：

```javascript
proxy: {
  '/api': {
    target: process.env.VUE_APP_API_BASE_URL || 'http://localhost:9090',
    // ...
  }
}
```

### 3. package.json

已添加更明确的打包脚本：

```json
{
  "scripts": {
    "serve": "vue-cli-service serve",
    "build": "vue-cli-service build",
    "build:dev": "vue-cli-service build --mode development",
    "build:prod": "vue-cli-service build --mode production"
  }
}
```

## ✅ 验证配置是否生效

### 开发环境验证

1. 运行 `npm run serve`
2. 打开浏览器控制台（F12）
3. 查看是否有环境配置日志输出
4. 检查网络请求，API请求应该指向 `http://localhost:9090/api/...`

### 生产环境验证

1. 运行 `npm run build`
2. 检查 `dist/js/app.xxx.js` 文件
3. 搜索 `VUE_APP_API_BASE_URL`，应该找不到（因为生产环境是空字符串）
4. 部署后，检查网络请求，API请求应该是相对路径 `/api/...`

## 🔍 环境变量规则

### 命名规则

- **必须以 `VUE_APP_` 开头**才能在代码中通过 `process.env` 访问
- 例如：`VUE_APP_API_BASE_URL` ✅
- 例如：`API_BASE_URL` ❌（无法访问）

### 访问方式

```javascript
// 在代码中访问环境变量
const apiUrl = process.env.VUE_APP_API_BASE_URL;
const env = process.env.VUE_APP_ENV;
```

### 环境文件优先级

```
.env.local > .env.[mode].local > .env.[mode] > .env
```

## ⚠️ 注意事项

1. **环境变量文件位置**：必须在项目根目录（与 `package.json` 同级）
2. **变量名格式**：必须以 `VUE_APP_` 开头
3. **修改后重启**：修改环境变量文件后，需要重启开发服务器或重新打包
4. **Git提交**：`.env.development` 和 `.env.production` 可以提交到Git，但 `.env.local` 不应该提交（通常包含敏感信息）

## 📊 工作流程

### 开发流程

```
1. 修改代码
2. npm run serve（自动使用 .env.development）
3. 代码热更新，自动代理到 localhost:9090
```

### 部署流程

```
1. npm run build（自动使用 .env.production）
2. 生成 dist 目录
3. 上传 dist 目录到服务器
4. Nginx 配置反向代理 /api -> 后端服务器
```

## 🎯 优势

1. **环境隔离**：开发和生产环境配置分离，互不干扰
2. **无需修改代码**：切换环境只需运行不同的命令
3. **配置集中管理**：所有环境变量集中在 `.env` 文件中
4. **易于维护**：修改配置只需编辑环境变量文件
